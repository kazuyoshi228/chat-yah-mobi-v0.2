# yah.mobile チャット デシジョンツリー設計書

## 分析結果サマリー

### 質問カテゴリ出現確率（10セッション・31ターン分析）

| カテゴリ | 出現率 | 平均RAGスコア | クレームリスク |
|---------|--------|-------------|--------------|
| 接続トラブル（繰り返し） | 51.6% | 0.315 | ⚠️ 高（未解決が続くと激化） |
| eSIMインストール・接続 | 25.8% | 0.517 | 中 |
| 返金・キャンセル | 9.7% | 0.442 | 🔴 最高（法的主張に発展しやすい） |
| 一般問い合わせ | 9.7% | 0.419 | 低 |
| 料金プラン | 3.2% | 0.357 | 低 |

### クレームリスク分析

**最高リスク: 返金・キャンセル**
- 「まだ使っていない」「例外にしてほしい」という主張が多い
- QRコード発行後は法的に返金不可だが、ユーザーは納得しにくい
- → 早期に「返金不可ポリシー」を明示し、代替案（トラブルシューティング）を提示することでクレーム化を防ぐ

**高リスク: 接続トラブル（繰り返し）**
- 「まだ繋がらない」「それでもダメ」の繰り返しでRAGスコアが0.157まで低下
- 解決策が尽きると「詐欺」「返金しろ」に発展しやすい
- → 3回以上同じ問題が続いたら即フォーム誘導（人間対応）

## デシジョンツリー構造

### Level 0: 開始（カテゴリ選択）

```
チャット開始
  ├─ 🔌 接続・設定トラブル（最多: 77.4%）
  ├─ 🛒 購入・プラン確認（3.2%）
  ├─ 💰 返金・キャンセル（9.7%）
  └─ 💬 その他・一般質問（9.7%）
```

### Level 1A: 接続・設定トラブル

```
接続・設定トラブル
  ├─ 📱 まだインストールしていない
  │    ├─ iPhone → [iPhone インストール手順を表示]
  │    └─ Android → [Android インストール手順を表示]
  │         └─ 解決した / 解決しない → [次のステップ or フォーム誘導]
  │
  └─ ✅ インストール済みだが繋がらない
       ├─ データローミングはONですか？
       │    ├─ いいえ → [データローミングON手順を表示] → 解決した / まだ繋がらない
       │    └─ はい → 次の質問へ
       │
       ├─ 機内モードのON/OFFを試しましたか？
       │    ├─ いいえ → [機内モードON/OFF手順を表示] → 解決した / まだ繋がらない
       │    └─ はい → 次の質問へ
       │
       ├─ APN設定は完了していますか？
       │    ├─ いいえ → [APN設定手順を表示] → 解決した / まだ繋がらない
       │    └─ はい → 次の質問へ
       │
       └─ 上記を全て試した → 🔴 [フォーム誘導: 専門サポートへ]
            （クレーム防止: 「お客様の状況を詳しく確認させていただきます」）
```

**クレーム防止ポイント**: 3ステップ以上試した場合は即フォーム誘導。「解決できない」と感じる前に人間対応に切り替える。

### Level 1B: 購入・プラン確認

```
購入・プラン確認
  ├─ 📋 プランを選びたい
  │    ├─ 滞在日数は？
  │    │    ├─ 1〜3日 → [ライトプラン 1GB/7日 ¥1,500 を表示]
  │    │    ├─ 4〜7日 → [スタンダードプラン 3GB/14日 ¥2,500 を表示]
  │    │    ├─ 8〜14日 → [プレミアムプラン 7GB/30日 ¥4,000 を表示]
  │    │    └─ 15日以上 → [アンリミテッドプラン ¥6,000 を表示]
  │    └─ 購入ページへのリンク
  │
  ├─ 💳 支払い方法を確認したい
  │    └─ [クレジットカード・PayPal・Apple Pay対応を表示]
  │
  └─ 📦 購入後の流れを確認したい
       └─ [QRコード発行→インストール手順の流れを表示]
```

### Level 1C: 返金・キャンセル（クレーム防止最重要）

```
返金・キャンセル
  ├─ QRコードはすでに発行されましたか？
  │    ├─ いいえ（未発行）→ [キャンセル手順を案内: 購入直後のみ可能]
  │    │
  │    └─ はい（発行済み）→ ⚠️ [重要説明を表示]
  │         「QRコード発行後は特定商取引法に基づき返金・キャンセルは
  │          原則としてお受けできません。」
  │         ├─ eSIMが使えない・繋がらない
  │         │    └─ [接続トラブルフローへ誘導]
  │         │         「まずeSIMが正常に動作するよう
  │         │          サポートさせていただきます」
  │         └─ それでも返金を希望する
  │              └─ 🔴 [フォーム誘導: 個別対応]
  │                   「お客様の状況を確認の上、
  │                    個別にご対応いたします」
  │                   （クレーム防止: 「断る」ではなく「確認する」）
```

**クレーム防止ポイント**: 
- 「返金できません」ではなく「原則としてお受けできません」
- 必ず代替案（トラブルシューティング）を先に提示
- フォーム誘導時は「個別対応」として希望を持たせる

### Level 1D: その他・一般質問

```
その他・一般質問
  ├─ 📡 対応エリア・キャリアについて → [日本全国・3キャリア対応を表示]
  ├─ 📱 対応機種について → [eSIM対応機種一覧リンク]
  ├─ ⏱️ 有効期限について → [プラン別有効期限を表示]
  └─ 💬 上記にない質問 → [AIチャットへ移行]
```

## フォーム誘導トリガー（クレーム防止）

以下の条件で即フォーム誘導:
1. 接続トラブルで3ステップ以上試した → 「専門サポートへ」
2. 返金希望（QRコード発行済み）→ 「個別対応」
3. ユーザーが怒り・不満を示す表現 → 即人間対応
4. 同じ問題を3回以上繰り返す → 自動フォーム誘導

## 実装方針

### データ構造（DB）
```typescript
chat_flow_nodes {
  id: string (e.g., "root", "connection", "connection_not_installed", ...)
  parentId: string | null
  type: "question" | "answer" | "redirect_form" | "redirect_ai"
  label: JSON (多言語: {ja, en, ko, zh, th, vi})
  content: JSON (多言語: 解決策テキスト)
  options: JSON (子ノードIDのリスト)
  formTrigger: boolean
  aiTrigger: boolean
  order: number
}
```

### UI設計
- チャット開始時: カテゴリ選択ボタン（4つ）を表示
- 選択後: 次の質問ボタンを表示（最大3つ）
- リーフノード: 解決策テキスト + 「解決した」「まだ解決しない」ボタン
- フォーム誘導: 既存のフォームリダイレクト機能を利用
- AIフォールバック: 「その他」選択 or 「解決しない」3回で自由入力AIチャットへ
